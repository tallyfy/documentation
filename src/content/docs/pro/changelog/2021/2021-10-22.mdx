### New Features

- **[API]** Support teams can now quickly find and review templates across all organizations with improved filtering options, making customer assistance faster and more efficient.
- **[API]** Faster task search powered by upgraded search technology that delivers instant results when looking for specific tasks.
- **[API]** You can now tag tasks and steps to organize your work better, plus filter your tasks and processes by tags to find what you need faster.

### Changes

- **[API]** Enhanced SSL certificate compatibility to ensure uninterrupted secure connections across all platform services.

### Bug Fixes

- **[API]** Improved API response times for checklist permissions to ensure consistently fast loading under all network conditions.
- **[API]** Fixed missing footers in daily digest emails to ensure consistent branding and legal compliance across all notifications.
- **[API]** Fixed tag adding errors that incorrectly claimed tags were already added when they weren't, making tag management more reliable.
- **[API]** Fixed an issue where the same tag could accidentally be added multiple times to a checklist, making tag organization cleaner and more reliable.
- **[API]** Fixed an issue where unapproved templates were showing up in the public library before admin review, ensuring only quality-checked templates are visible to users.
- **[API]** Fixed one-off tasks getting disconnected from templates when template versions changed, ensuring tasks stay properly linked.
- **[API]** Fixed "Page not found" errors that occurred when archiving templates, making the archive process more reliable across all template types.
- **[API]** Email notification links from guest comments now properly open the specific task instead of redirecting to your dashboard homepage.

### Removed

- **[API]** Manual task assignment has been streamlined by removing the self-assignment option for unassigned users, creating clearer workflow responsibilities.

### Performance

- **[API]** Fixed API response times during high traffic periods by optimizing how we handle traffic spikes, ensuring smooth performance even when usage surges.
