### Changes

- **[UI]** Cleaned up permissions sidebar by removing redundant "Permissions" label to make the interface cleaner and easier to read.
- **[UI]** Step deletion now shows exactly which step is blocking removal and links directly to fix the timing conflict, eliminating confusion.
- **[UI]** You can now auto-archive completed processes after 1, 2, 3, or 5 days for better control over short-term cleanup.
- **[UI]** Process cards in Board View now show full template names and fit more details by redesigning the layout with better spacing and contrast.
- **[UI]** Made automation trigger labels clearer by changing "is completed" to "is sent" for email tasks, making it easier to understand when your automations will run.

### Bug Fixes

- **[UI]** Email notifications are now working reliably again, ensuring you receive all task assignments and updates without delay.
- **[UI]** Guest users now properly receive email notifications with recovery links when they need to regain access to their account.
- **[UI]** Fixed password reset emails not being delivered when using the "Forgot Your Password" option, making account recovery reliable again.
- **[UI]** Fixed email delivery issue so new member invitations now arrive reliably in inboxes.
- **[UI]** Fixed text display issue where letters like g, j, p, q, y were getting cut off in task titles, ensuring clean and readable text throughout your workflows.
- **[UI]** Fixed radio button selections in forms to properly show which option you picked - your choices were always saved but weren't visually highlighted.
- **[UI]** Fixed text display where letters like 'g' and 'y' were getting cut off at the bottom, making task numbers and text easier to read.
- **[UI]** Fixed automation setup errors in templates when using dropdown, checklist, or radio fields with "is" or "is not" conditions.
- **[UI]** Fixed template creation getting stuck when users forgot to add a name - now the Generate button stays disabled until both name and file are provided.
- **[UI]** Fixed task counts in milestones to update instantly when hiding, showing, or linking tasks without needing to refresh the page.
- **[UI]** Fixed duplicated steps and tasks to appear in the correct position immediately without needing to refresh the page.
- **[UI]** Fixed tooltip display in the launch wizard so assignee names are now fully visible and scrollable without being blocked by UI elements.
- **[UI]** Template duplication now copies webhooks and tags automatically, eliminating the need to manually recreate automations in your duplicated templates.
- **[UI]** New tasks now appear instantly in document views without needing to refresh your browser, and they show up in the right position at the bottom.
- **[UI]** Fixed menu positioning and save button behavior when customizing error pages, making the editor easier to use in both normal and fullscreen modes.
- **[UI]** Fixed visual glitch where the complete button was cut off for tasks requiring all assignees to finish.
