### New Features

- **[UI]** Added a "Force sequential" toggle that automatically shows workflow steps one at a time, eliminating the need to create separate visibility automations for sequential processes.
- **[API]** Light role members now see a helpful upgrade message when they try to edit templates, making it easier for them to request the right permissions.
- **[API]** Support teams can now instantly sync subscription data instead of waiting up to 24 hours, ensuring faster resolution of billing-related customer inquiries.
- **[API]** You can now edit role assignments directly from the Tasks view and active processes using new pencil icons, making it faster to reassign work without navigating elsewhere.

### Changes

- **[UI]** Added step nesting with milestones, inline step creation, and duplicate step options to make building complex procedure templates much faster and easier.
- **[UI]** Client reports now feature updated styling with improved typography, table formatting, and visual design that matches our latest branding guidelines.
- **[UI]** Updated all help links throughout the app to point to our improved documentation pages for faster, more reliable access to guides.
- **[API]** Stage titles can now be up to 250 characters long, giving you more space for descriptive workflow names.
- **[API]** Updated all help links to direct you to our improved documentation website for faster access to current guides.

### Bug Fixes

- **[UI]** Fixed step and task numbers to update instantly when moving items in and out of milestones, eliminating the need for page refreshes.
- **[UI]** Fixed duplicate "Congratulations! You've completed this process" messages that were cluttering the screen after finishing a process.
- **[UI]** Milestone labels now support their full 250-character limit instead of being cut off at 31 characters, giving you more space for descriptive names.
- **[UI]** Fixed real-time automation display issue when automation rules involved tasks within milestones, eliminating the need for manual page refreshes.
- **[UI]** Fixed checklist items disappearing when checked - single characters and short text now stay visible in page view.
- **[UI]** Rich text fields in document templates now stay editable when you reopen completed tasks, eliminating the need to refresh your page.
- **[UI]** Fixed milestone loading issues so empty milestones now display correctly instead of showing endless loading spinners, plus task counts update instantly when moving tasks.
- **[UI]** Fixed AI template generation failing for new organizations and trial accounts, plus search now works properly when typing 3+ characters for industry types.
- **[UI]** Fixed "Launch another process" automation settings reverting unexpectedly and resolved permission errors when completing tasks that trigger new processes.
- **[UI]** Fixed step numbering delays and visibility issues when working with milestones to make organizing your process steps smoother.
- **[UI]** Fixed Light role users incorrectly appearing in template permission dropdowns, ensuring proper role restrictions are maintained.
- **[UI]** Fixed attachment viewing in archived processes and improved hyperlink display in form fields, so you can access files and see properly formatted text without extra steps.
- **[UI]** Fixed step ordering in procedure templates so steps now stay in the correct sequence immediately after deleting a milestone, without needing to refresh the page.
- **[UI]** Fixed step reordering in templates with milestones so your process tasks now stay in the correct order after launch.
- **[UI]** Fixed user deactivation when using "re-assign all tasks and steps" option, eliminating incorrect error messages that previously blocked the process.
- **[UI]** Fixed assignee display issues in description boxes so team members and guests now show consistently with proper formatting instead of error messages or broken links.
- **[API]** Fixed a bug where updating task details would sometimes fail due to improved data validation in the background.
- **[API]** Fixed auto-naming for processes launched from task completion so the template name now displays correctly instead of showing just a hyphen.
- **[API]** Fixed an issue where removing guest users would fail when they had completed tasks, ensuring smooth guest management.
- **[API]** Template duplication now properly copies all webhook URLs, ensuring your integrations work seamlessly in duplicated templates.

### Deprecated

- **[UI]** Temporarily keeping the Form template filter option available in preparation for upcoming form template functionality enhancements.

### Removed

- **[UI]** Cleaned up leftover form template references throughout the platform to avoid confusion since this feature was discontinued.

### Performance

- **[UI]** Made user selection and loading much faster for organizations with thousands of members by fixing memory issues and reducing unnecessary data requests.
