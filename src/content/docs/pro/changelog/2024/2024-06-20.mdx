### New Features

- **[UI]** Support admins can now selectively remove trial limits for specific organizations, giving more flexibility to help customers evaluate Tallyfy properly.
- **[UI]** Light role users can now see a helpful message with upgrade options when they try to create or edit templates, making role management clearer.

### Bug Fixes

- **[UI]** Fixed dropdown display issues in admin settings so email digest scheduling now shows readable day options instead of confusing code text.
- **[UI]** Fixed email addresses and other text showing duplicated characters in task descriptions, ensuring all form field content displays correctly.
- **[UI]** Fixed custom permissions to properly default to admin access when left blank, instead of incorrectly reverting to "Everyone" permissions.
- **[UI]** Fixed form field navigation to ensure dropdown, checklist, radio button, and table fields are properly configured with required options before you can continue building your template.
- **[UI]** Fixed duplicate checklist form fields that appeared when switching between steps during incomplete form creation.
- **[UI]** Fixed milestone positioning and numbering to update instantly when deleted or moved, plus guest users can now properly see their assigned milestone tasks.
- **[UI]** Tasks now properly appear when filtering by tags, including tasks from tagged processes that were previously missing from search results.
- **[UI]** Fixed Tracker exports to show correct user names and progress values instead of blank cells and incorrect date formatting.
- **[UI]** Fixed Spanish translation issues across forms, deadlines, and activity columns by moving text from code to proper language files for accurate display.
- **[UI]** Fixed cursor styling on template and automation dropdowns so they now show a pointer when hovering, making it clearer they're clickable.
- **[UI]** Fixed missing menu options on completed tasks so guests can now properly access their Forever Links and view finished work.
