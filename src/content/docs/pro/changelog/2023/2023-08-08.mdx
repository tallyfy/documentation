### New Features

- **[UI]** You can now favorite processes directly from the Activity Panel with a simple star click, making it easier to prioritize important work.
- **[UI]** Added system variables (process ID, task ID, field ID) to text editor dropdowns for easier reference in forms and descriptions.
- **[UI]** Added support for Light user roles with simplified billing, letting you invite team members at a lower cost while maintaining full control over who can edit templates.
- **[UI]** Form fields now support default values to pre-fill content and guide users, making forms faster to complete and reducing errors.
- **[UI]** You can now add prefix and suffix text (like "$" or "units") to form fields, making it clearer what information to enter.

### Changes

- **[UI]** Tallyfy AI now automatically uses your organization's industry setting when creating templates, making the setup process faster and more personalized.
- **[UI]** Improved form field variable insertion by streamlining how the system identifies tasks, making the feature work more efficiently across all user interfaces.
- **[UI]** Form fields now load faster by fetching only the data you need for each specific process instead of loading everything.
- **[UI]** Document editor now highlights the Field Controls area with background coloring, making it clearer which field you're editing.
- **[UI]** Tables now support column sorting for custom fields and filtering by form field values, making it easier to find and organize your data.
- **[UI]** Moved tag and folder management to the main view with handy dropdowns, making it much easier to organize and find your content.

- **[UI]** Fixed missing close buttons in automation trigger sections so you can now properly close trigger fields when setting up workflow automations.

- **[UI]** Fixed signup, password reset, and task completion issues that were preventing users from receiving important email notifications.
- **[UI]** Fixed auto-naming dropdown to instantly show newly added required fields from Key Order Forms without needing to refresh the page.
- **[UI]** Fixed emoticons now displaying consistently when switching between normal and fullscreen editing modes, providing a smoother writing experience.
- **[UI]** Fixed file uploads so documents now download properly and display correctly without corruption errors.
- **[UI]** Fixed image and video uploads in rich text editor that were failing with loading errors, restoring full media functionality.
- **[UI]** Fixed error pages briefly flashing in Tracker Table view, making the page load smoothly without visual interruptions.
- **[UI]** Fixed horizontal scrolling glitch that occurred when clicking the Sort menu in Tracker view for smoother navigation.
- **[UI]** Fixed guest links that sometimes failed to load when accessing processes, ensuring reliable access for external collaborators.
- **[UI]** Fixed sorting and filtering for dropdown, checklist, and radio form fields in Tracker's Table view for easier process data organization.
- **[UI]** Fixed guidance content in template steps now saves automatically when you click away from the field after editing.
- **[UI]** Fixed inconsistent help text display in document template builder so guidance appears properly for all field types.
- **[UI]** Fixed list bullets and numbers not showing when using the text editor in fullscreen mode.
- **[UI]** Library procedure alerts now show clear validation messages instead of confusing error text when default choices aren't selected.
- **[UI]** Fixed multiple template editor issues including auto-save failures that caused data loss, overly long auto-generated names, and date field settings not saving properly.

- **[UI]** Fixed search for guest users when assigning tasks, so you can now easily find and assign existing guests in all areas.

- **[UI]** Fixed animated GIFs incorrectly appearing in error messages when custom error settings were enabled across Tracker, Tasks, and template search views.
- **[UI]** Fixed document editor fullscreen mode so form fields, menus, and controls now display properly and respond correctly when editing content.
- **[UI]** Fixed snippet search in Personalization settings so you can now find your saved snippets by typing keywords in the dropdown.
- **[UI]** Standard users can now view the activity feed in the Blueprint Library, showing updates for all templates they have access to.
- **[UI]** Fixed form display issues and added smart auto-naming to make launching processes smoother with more consistent names.
- **[UI]** Improved checklist form fields with clearer validation options and fixed alignment, error messages, and text display issues for a smoother editing experience.
- **[UI]** Fixed text overflow issue where long template descriptions would spill outside task borders, making content easier to read.
