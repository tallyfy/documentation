### New Features

- **[UI]** Added group and job title filters under Assignee in the Tasks sidebar, making it easier to find tasks assigned to specific teams or roles.
- **[UI]** New "Assignee Picker" form field lets you select team members and guests in forms for smoother task assignment workflows.

### Changes

- **[UI]** Released interface improvements and API enhancements after thorough quality testing to ensure a smoother user experience.
- **[UI]** Simplified navigation by renaming "Blueprint" to "Template" and "Library" to "Templates" throughout the app for clearer, more intuitive labeling.
- **[UI]** You can now assign multiple owners to templates and processes instead of being limited to just one, making collaboration easier.
- **[UI]** Added a toggle to reverse automation rules when reopening specific tasks, giving you more control over your workflow execution.

### Bug Fixes

- **[UI]** Fixed Universal Search so clicking on snippet results now properly opens the specific snippet instead of redirecting to the wrong page.
- **[UI]** Improved form field label display validation to ensure all text shows properly when launching processes, even with maximum character lengths.
- **[UI]** Clear error messages now explain exactly why template steps can't be deleted when they're connected to automations, preventing confusion and accidental workflow breaks.
- **[UI]** Added stability improvements to the folder navigation scroll button to prevent occasional disappearing on ultra-wide displays.
- **[UI]** Fixed misaligned member and guest icons in the Launch Wizard's final review page for a cleaner, more professional appearance.
- **[UI]** Template folder deletions now show the folder name in activity logs, making it easier to track what was deleted and by whom.
- **[UI]** Fixed unassigned tasks now correctly go to the process launcher instead of incorrectly defaulting to the template creator.
- **[UI]** Fixed automation actions displaying in wrong order after first template launch when using 4+ actions in a single rule.
- **[UI]** Fixed checklist fields to save your progress even when all items aren't checked yet, so you won't lose work when returning later.
- **[UI]** Fixed automation conditions so "AND" rules now properly require all conditions to be met instead of incorrectly behaving like "OR" rules.
- **[UI]** Fixed spelling of "Professional Services" in AI template creation to ensure proper formatting and consistency.
- **[UI]** Fixed dropdown filters in Tasks and Tracker views that were overlapping with other elements and showing incorrect filter options when switching between different filter types.
- **[UI]** Fixed confusing timing settings in procedure templates by removing disabled "before" options when launch is selected, making automation setup clearer.
- **[UI]** Fixed table form fields, rich text formatting, and kick-off field displays when using Insert Variable in task summaries.
- **[UI]** You can now start with just 1 member instead of being forced to pay for 4 minimum members, making Tallyfy more affordable for smaller teams.
- **[UI]** Fixed tooltip positioning in the document editor's fullscreen mode so all menus and hints now appear in the right place.
- **[UI]** Added sorting by creation date for both members and guests tables, making it easier to find your newest or oldest team members.
