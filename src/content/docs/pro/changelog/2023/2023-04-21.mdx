### New Features

- **[UI]** New users now see a helpful one-day banner explaining Tasks, Tracker, and Library sections to get started faster.
- **[UI]** You can now upload your original flowcharts, documents, or forms when creating templates to help train our AI for better automation suggestions.
- **[UI]** Added default value support for radio buttons, dropdowns, and checkboxes so you can pre-select common choices and save time filling out forms.
- **[UI]** Added default content for text fields so you can pre-fill forms with helpful starter text or dynamic variables.
- **[API]** Login page now shows a personalized greeting based on your location, making your first impression more welcoming and relevant.
- **[API]** You can now upload and attach files directly to your templates, with support for documents, images, and spreadsheets to keep everything organized in one place.
- **[API]** Admins can now view a complete history of how each team member has signed in over time for better security oversight.

### Changes

- **[UI]** Verified that our member email uniqueness system works correctly, ensuring each team member has a distinct account for better security and organization.
- **[UI]** Fixed document editor formatting issues including page breaks, headers, tables, and bullet points for cleaner document creation.
- **[UI]** Template steps now display helpful icons so you can instantly see what type of step it is without needing extra labels.
- **[UI]** Improved task organization planning to group tasks by process in the Tasks view, making it easier to see related work together.
- **[UI]** Template library now shows how many templates are in each folder, making it easier to organize and find your processes.
- **[API]** Improved email reply addresses to use a cleaner, more consistent format across all system notifications.
- **[API]** Activity feeds now update every 10 seconds instead of 5 minutes, so you see changes to processes and documents almost instantly.
- **[API]** Made login, signup, and password reset pages use consistent fonts that match the main app for a cleaner, more professional look.
- **[API]** You can now set a default folder for new processes and control whether users can change it, making process organization more streamlined and consistent.
- **[API]** Organization invites now work seamlessly for both internal SSO users and external guests with separate acceptance options in each invite email.
- **[API]** Verified that default templates are properly created once during organization setup, ensuring a clean starting workspace.

### Bug Fixes

- **[UI]** Fixed a temporary issue where document read view wasn't working properly in Tracker and Library, ensuring smooth document access.
- **[UI]** Improved visibility of automation trigger labels so they're clearly readable with all branding color themes.
- **[UI]** Document library now shows all templates by default instead of hiding newly created documents behind filters.
- **[UI]** Fixed document field removal to correctly delete the intended field rather than deleting by position, preventing accidental removal of wrong fields when rearranging.
- **[UI]** Fixed signup email links that were incorrectly redirecting new users to login instead of account creation.
- **[UI]** Fixed document form fields throwing errors after deletion and page refresh, ensuring smooth editing of document templates.
- **[UI]** Fixed accordion panels in forms that would sometimes get stuck open, making them easier to navigate and use.
- **[UI]** Fixed navigation in the Library view so users can now properly access "Archived" and "Activity" views instead of being stuck on "Uncategorized" only.
- **[UI]** Fixed duplicate update errors when filling out document template fields that previously caused "Unable to process request" messages.
- **[UI]** SSO login now shows clear details about who disabled your account and when, making it easier to understand why you can't access Tallyfy.
- **[UI]** Fixed automation rules for short text fields to show proper criteria options like "contains" and "is empty" instead of just "Select".
- **[UI]** Fixed processes unexpectedly opening when archiving them in Tracker view, making bulk archiving much smoother and faster.
- **[UI]** Document field updates now display more predictably in the preview, making it clearer where your changes appear in the final document.
- **[UI]** File uploads now support more formats including .tif, .bmp, and .xslt files, so you can upload any process document without extension errors.
- **[UI]** Fixed magic link formatting issues with dropdown and checklist fields by clarifying the correct syntax through better examples and documentation.
- **[UI]** Confirmed that job title suggestions in the workflow builder show all roles across your organization, ensuring maximum reusability when assigning tasks.
- **[UI]** Fixed form field options in templates so you can now properly delete empty dropdown, checklist, and radio button choices without accidentally removing the wrong ones.
- **[UI]** Fixed notification links for out-of-process tasks so clicking email or in-app notifications now properly shows and opens the mentioned task.
- **[API]** Fixed email notifications for @mentions to show only one working link, eliminating confusion and navigation errors.
- **[API]** Fixed social login authentication with Google and Microsoft to work more reliably during sign-in and signup.
- **[API]** Fixed login error messages to properly display when incorrect credentials are entered, making it clearer when you need to reset your password.
- **[API]** Fixed welcome email sign-in links so invited members can now properly access their new organization after clicking the button.
- **[API]** Fixed a system error that could prevent creating new automated recurring jobs, ensuring smoother workflow automation setup.
- **[API]** Fixed task overdue labels so they now accurately show when tasks are actually behind schedule instead of incorrectly flagging manual tasks.

### Removed

- **[UI]** After reviewing feedback, we strengthened our commitment to consistent company branding across all document templates for a more professional experience.

### Security

- **[API]** Made webhook URL validation more secure by replacing external requests with internal checks, preventing potential security risks while maintaining reliable automation setup.
- **[API]** Improved API security by adding better validation to prevent potential abuse when managing user preferences.
