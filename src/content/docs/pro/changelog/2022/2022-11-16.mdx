### New Features

- **[UI]** Support teams can now permanently remove orphaned users directly through the interface with a two-step confirmation process for safer user management.

### Changes

- **[UI]** Comments now show all types by default instead of hiding auto-generated ones, so you won't miss important updates.
- **[UI]** Enhanced quality assurance testing completed to ensure smoother performance and reliability for all users.
- **[UI]** Made the upgrade path clearer for trial users by simplifying the billing page and adding a prominent "Upgrade" button to the main menu.
- **[UI]** Template creation wizard now displays larger with better spacing and clearer labels, making it easier to browse and select templates.
- **[UI]** Document template heading styles now follow a cleaner hierarchy with improved sizing and consistent black text formatting for better readability.
- **[UI]** Auto-loading now displays all your tasks and processes without needing to click "Show more" buttons repeatedly.

### Bug Fixes

- **[UI]** Fixed document template content being lost when adding form fields to ensure your work is always preserved.
- **[UI]** Launch summary now correctly shows all people assigned to roles instead of only displaying the launcher.
- **[UI]** Fixed kick-off form fields used for process auto-naming to stay required, preventing template setup conflicts and ensuring consistent process naming.
- **[UI]** Fixed the Tags filter in Custom Views to correctly show tags instead of people, ensuring all filters now display the right values.
- **[UI]** Fixed floating buttons that were blocking the comments area in approval tasks, making it easier to add comments and feedback.
- **[UI]** Config settings in the template editor now load instantly instead of taking several seconds to open.
- **[UI]** Fixed member invite form to show clear error messages when first or last names exceed 32 characters, preventing silent failures during signup.
- **[UI]** Fixed assignee names now properly showing when completing group tasks where all members must complete the task.
- **[UI]** Fixed an issue where disabled users would still appear as task creators after reassignment, ensuring cleaner user management when removing team members.
- **[UI]** Fixed URL links in workflow variables so they're now properly clickable when referencing text from previous steps.
- **[UI]** Fixed task completion buttons for group assignments so all members can see who's finished and buttons work properly for everyone.

### Removed

- **[UI]** Removed source maps from production to better protect code security while maintaining full error tracking capabilities.
