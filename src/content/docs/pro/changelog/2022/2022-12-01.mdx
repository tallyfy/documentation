### Changes

- **[API]** Optimized our monitoring setup to reduce unnecessary network traffic while maintaining reliable uptime checks for better performance.
- **[API]** Upgraded database infrastructure to faster ARM64 architecture, improving overall system performance and reliability.

### Bug Fixes

- **[API]** Fixed login redirects that were getting stuck in loops, so you can now access your account smoothly without redirect errors.
- **[API]** Fixed a service disruption that briefly made certain features unavailable, ensuring consistent access across all environments.
- **[API]** Fixed automated data collection from incorrectly updating guest activity timestamps, ensuring more accurate reporting of when guests actually use Tallyfy.
- **[API]** Fixed signup process failing at the final step, ensuring new accounts are created successfully without errors.
- **[API]** Fixed task filtering by checklist so the API now correctly returns all matching tasks instead of showing empty results.
- **[API]** Process names now display cleanly without HTML formatting when using auto-naming with blueprint names.
- **[API]** Fixed an intermittent error that occasionally prevented adding explanation videos to document templates, making the feature more reliable.
- **[API]** Fixed checklist updates failing when changing certain properties, ensuring all checklist modifications now save properly without errors.
- **[API]** Fixed template duplication to correctly preserve admin-only permissions, preventing access errors and ensuring permission displays match actual restrictions.
- **[API]** Fixed activity feed display errors that prevented some users from viewing their recent process updates and notifications.
- **[API]** Fixed a bug that prevented form field creation and editing in templates, ensuring smoother template building workflow.
- **[API]** Fixed folder navigation errors that were preventing users from accessing their organized content properly.
- **[API]** Added missing documentation for the API endpoint that searches processes by kickoff value, making it easier for developers to discover and use this feature.
- **[API]** Fixed a database error that prevented the system from properly handling user-specific data requests, improving reliability when viewing your assigned content.
- **[API]** Fixed invited users getting stuck when joining SSO-enabled organizations - they can now login seamlessly using either the email invite or normal SSO.
- **[API]** Fixed table fields in templates now properly save your data when launching processes, ensuring all your information is captured correctly.
- **[API]** Fixed a rare error that could occur when viewing activity history, ensuring all process activities display correctly without interruption.
- **[API]** Fixed API responses to use correct domain names in all links, preventing redirect issues and improving reliability when accessing Tallyfy through different network routes.

### Security

- **[API]** Login errors now show a single helpful message instead of revealing whether an email exists, improving security while guiding you to reset passwords when needed.

### Performance

- **[API]** Made organization loading faster by removing unnecessary database lookups that were slowing down the app.
- **[API]** We improved system reliability by adding redundant servers, ensuring your workflows stay running even during unexpected outages.

### Dependencies

- **[API]** Updated error tracking system to the latest version for improved reliability and better issue detection across the platform.
