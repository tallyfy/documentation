### New Features

- **[UI]** Added a simple filter button to temporarily hide completed tasks in processes, making it easier to focus on what still needs to be done.
- **[UI]** Process launchers can now be required to organize new processes with folders and tags, making it easier to keep your workspace tidy and organized.
- **[UI]** You can now add explanation videos to your templates from YouTube, Vimeo, or Loom to help team members understand processes better.
- **[API]** Added automated backups for guest accounts, templates, and code snippets so you can easily access and export all your workflow data.

### Changes

- **[UI]** Auto-naming for processes now works consistently with form fields regardless of their required status, preventing naming conflicts.
- **[UI]** We moved the "Publicize" settings into the "Permissions" tab so all access controls are now in one place for easier setup.
- **[UI]** Simplified the billing page for trial users with clearer upgrade options and a prominent green upgrade button to make purchasing easier.
- **[UI]** Simplified the "Launch Process" wizard header to just say "Launch" for a cleaner, more focused interface.
- **[UI]** Changed "Launch Document" button to say "Use Document" which better matches how people naturally talk about working with documents.
- **[UI]** Process auto-naming now works with required Date, Dropdown, Radio, and Checklist fields, making it easier to create meaningful process names automatically.
- **[UI]** Added specific date option to Launch Process power tools for more flexible deadline management, matching existing functionality.
- **[UI]** Redesigned empty folder view with a prominent "Create Blueprint" button to make starting new templates faster and more intuitive.
- **[API]** Simplified email notifications when removing team members by using clearer, consistent terminology that matches the rest of the platform.
- **[API]** Improved file storage flexibility by making storage backends easily swappable for future geographic compliance and cost optimization.

### Bug Fixes

- **[UI]** Fixed kick-off form fields used for auto-naming processes to stay required, preventing confusion when launching new processes.
- **[UI]** Fixed process auto-naming to only show relevant field types, removing large text fields that shouldn't appear as naming options.
- **[UI]** Fixed launch wizard showing unnecessary role assignment page when no steps had roles, making the process launch flow cleaner and more intuitive.
- **[UI]** Fixed intermittent freezing issues where task panes wouldn't close and templates couldn't launch after extended use.
- **[UI]** The "Assigned to a member" filter in Tasks view now responds properly when selecting team members to filter tasks.
- **[UI]** Fixed timezone dropdown menus in Settings that weren't showing options, so you can now properly select your timezone in both Profile and Organization sections.
- **[UI]** Fixed "Completed by" field now properly displays when tasks assigned to groups are completed, improving task tracking clarity.
- **[UI]** Fixed apostrophes in table column names that previously caused errors when configuring data capture steps in templates.
- **[UI]** Improved spacing around the "You have no tasks" message in the guest view for better readability and visual clarity.
- **[UI]** Fixed double-clicking to select all text in input fields, which stopped working properly after a recent interface update.
- **[UI]** Fixed an issue where guest users could accidentally become the only person assigned to a task - now organization members are always included as assignees.
- **[UI]** Fixed empty state message in Documents filter view so you see proper text instead of error codes.
- **[UI]** Fixed confusing comment errors on public sample templates by properly hiding the comment button when commenting isn't available.
- **[UI]** Fixed SSO login loops caused by incorrectly configured identity provider URLs, ensuring smooth authentication for all users.
- **[UI]** Templates moved to Uncategorized folders now automatically unpin themselves, eliminating sorting confusion and maintaining consistent folder behavior.
- **[UI]** Fixed redundant email reminders being sent to guests after completing each task, reducing notification spam while keeping daily digest emails.
- **[UI]** Fixed dropdown menu alignment so text and icons now have proper spacing and look consistently aligned across all menus.
- **[UI]** Disabled users now see a clear message with date and administrator details instead of confusing error pages when trying to log in.
- **[UI]** Fixed form field labels getting cut off in the Process Launch Wizard so you can now see complete field names when launching processes.
- **[UI]** Fixed an issue where one-off task creator assignments weren't properly cleared when disabling users, making the process smoother and more reliable.
- **[UI]** Fixed duplicate template creation when clicking create buttons multiple times by preventing accidental double-clicks during the process.
- **[UI]** Fixed Universal Search to properly show restricted tasks by redirecting you to the assignee's filtered view instead of your empty Tasks page.
- **[UI]** Standard users now see clearer controls in Snippets management - edit options are hidden when you don't have permission to change them.
- **[UI]** Blueprints now instantly disappear from your favorites list when you unfavorite them, keeping your view clean and up-to-date.
- **[UI]** Group task completion now shows completer names and updates the complete button correctly, matching the clear progress tracking you get with individual assignments.
- **[UI]** Fixed missing annotation details in the task pane and now properly displays your timezone instead of UTC for all annotation timestamps.
- **[UI]** Enhanced task filtering so you can now see group-assigned tasks when filtering by individual team members who belong to those groups.
- **[UI]** Support administrators can now edit the visibility settings of public templates across all organizations, improving template management capabilities.
- **[API]** Fixed a database error that prevented saving organization data when names exceeded length limits, ensuring all organization information is properly stored.
- **[API]** Fixed a data storage issue that was preventing some organization settings from being saved properly.
- **[API]** Fixed an issue where organizing items into folders could sometimes fail due to duplicate entries, making folder management more reliable.
- **[API]** Fixed activity feed loading errors that could occur when viewing process updates and deadline changes.
- **[API]** Fixed activity feeds to properly group automatic task comments under their parent process instead of showing them separately.
- **[API]** Fixed tasks with only guest assignees disappearing from your task views by automatically including you as an assignee to keep them visible.
- **[API]** Fixed email notifications timing out when task updates were sent, ensuring you now receive timely alerts about task changes.
- **[API]** Fixed activity feed comments showing incorrect information when launching new processes from approval tasks, ensuring accurate tracking history.
- **[API]** Fixed an error that prevented adding team members to organizations when duplicate user relationships already existed.

### Deprecated

- **[UI]** Added ability to disable guest users and make their links inaccessible, just like you can with team members.

### Removed

- **[UI]** Removed video recording analytics to enhance user privacy and streamline our application for better performance.

### Security

- **[UI]** Only administrators can now permanently delete processes and templates, protecting your important workflows from accidental removal.

### Performance

- **[API]** API response times improved by identifying optimization opportunities for throttle management and maintenance checks.
